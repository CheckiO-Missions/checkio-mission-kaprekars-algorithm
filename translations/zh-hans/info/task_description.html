让我们随机取一个正整数（如 5914），然后应用Kaprekar's routine ：首先 首先，按数字降序（9541）和升序（1459）创建两个新整数。然后 用较大的数字减去较小的数字（9541 - 1459 = 8082）。对 新数字 8082。

对任何正整数应用卡普雷卡尔算法都会产生三种情况：

  1. 达到恒整数
    对于 5914，数字序列是5914, 8082, 8532, 6174, 6174, 6174, ...
    因此，6174 是与 5914 相对应的卡普雷卡尔常数。
  2. 达到整数循环
    例如48, 36,27, 45, 09, 81, 63, 27 , 45, 09, 81, 63, 27, 45, 09, 81, 63,...
    (用两种不同颜色突出显示卡普雷卡尔圆 27、45、09、81、63）
  3. 达到零点
    尽管从技术上讲，零也是一个常数（如 7、0、0、0、......），但数学家并不认为零是一个卡普雷卡尔常数。 是卡普雷卡尔常数。

你可能已经注意到了，前导零可以发挥重要作用：在循环27 中，前导零是一个常数（例如，7、0、0、0、...）、 45，09，81，63 的循环中，9 必须被视为 09，而不仅仅是 9。对于任何中间结果，请考虑 将输入整数的总位数填满前导零。

输入正整数n(0 ≤n ≤999.999)

输出：由三个值组成的元组

  1. 一个整数，它应该是n 的卡普雷卡尔常数，或者（在循环的情况下）从n 开始输入的卡普雷卡尔循环的第一个数字。 的第一个数字。
    说明尽管 48 和 92 这两个整数通向同一个圆（即 27、45、09、81、63），但这两个整数进入这个圆的位置不同、 这两个整数进入这个循环的数字是不同的。因此，您的函数应返回 27（n= 48）和 63（n= 92）。 对于n= 92。
  2. 必须对n进行多少次 Kaprekar 运算，直到达到一个常数或循环为止。如果n 已经是一个 Kaprekar 常量（或 Kaprekar 循环中的一个数字），则操作次数为零。
  3. 布尔值或 None 表示n 发生了哪种情况：情况 A 为 True，情况 B 为 False，情况 C 为 None。

示例 

assert kaprekar_algorithm(5914) == (6174, 3, True)
assert kaprekar_algorithm(6174) == (6174, 0, True)
assert kaprekar_algorithm(48) == (27, 2, False)
assert kaprekar_algorithm(27) == (27, 0, False)
assert kaprekar_algorithm(45) == (45, 0, False)
assert kaprekar_algorithm(111) == (0, 1, None)
assert kaprekar_algorithm(9) == (0, 1, None)
