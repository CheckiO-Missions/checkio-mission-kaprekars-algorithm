Prenons un nombre entier positif aléatoire (par exemple 5914) et appliquons-lui la méthode Kaprekar's routine: Dans un premier temps, vous créez deux nouveaux entiers d'abord, vous créez deux nouveaux entiers en classant les chiffres par ordre décroissant (9541) et croissant (1459). Ensuite vous soustrayez le plus petit du plus grand (9541 - 1459 = 8082). Répétez cette procédure pour le nouveau nombre 8082.

L'application de l'algorithme de Kaprekar à n'importe quel nombre entier positif peut donner lieu à trois cas de figure:

  1. Un nombre entier constant est atteint
    Pour 5914, la séquence de nombres est la suivante 5914, 8082, 8532, 6174, 6174, 6174, ...
    Ainsi, 6174 est la constante de Kaprekar correspondant à 5914.
  2. Un cycle d'entiers est atteint
    Exemple: 48, 36, 27, 45, 09, 81, 63, 27, 45, 09, 81, 63, 27, 45, 09, 81, 63, ...
    (Deux couleurs différentes sont utilisées pour mettre en évidence le cercle de Kaprekar 27, 45, 09, 81, 63)
  3. Le zéro est atteint
    Même si techniquement le zéro est aussi une constante (par exemple 7, 0, 0, 0, ...), les mathématiciens ne considèrent pas le zéro comme une constante de Kaprekar comme une constante de Kaprekar.

Vous avez peut-être remarqué que les zéros initiaux peuvent jouer un rôle important: dans le cycle 27, 45, 09, 81, 63, le neuf doit être traité comme 09, et pas seulement comme 9. Pour tout résultat intermédiaire, envisagez de de remplir de zéros en tête le nombre total de chiffres de l'entier d'entrée.

Entrée: Un entier positif n (0 ≤ n ≤ 999.999)

Sortie: Un tuple composé de trois valeurs:

  1. Un entier qui doit être soit la constante de Kaprekar à n, soit (dans le cas d'un cycle) le premier numéro d'un cycle de Kaprekar entré à partir de n Cycle de Kaprekar qui est entré à partir de n.
    Note de clarification: Même si les deux nombres entiers 48 et 92 mènent au même cercle (c'est-à-dire 27, 45, 09, 81, 63), les deux entiers entrent dans ce cercle à des numéros différents. Ainsi, votre fonction devrait retourner 27 (pour n = 48) et 63 pour n = 92.
  2. Le nombre d'opérations de Kaprekar qui doivent être appliquées à n jusqu'à ce qu'une constante ou un cycle soit atteint. Si n est déjà une constante Kaprekar (ou un nombre d'un cycle Kaprekar), cela doit être compté comme zéro opération.
  3. Un booléen ou None indiquant quel cas s'est produit pour n: True pour le cas A, False pour le cas B, None pour le cas C.

Exemples: 

assert kaprekar_algorithm(5914) == (6174, 3, True)
assert kaprekar_algorithm(6174) == (6174, 0, True)
assert kaprekar_algorithm(48) == (27, 2, False)
assert kaprekar_algorithm(27) == (27, 0, False)
assert kaprekar_algorithm(45) == (45, 0, False)
assert kaprekar_algorithm(111) == (0, 1, None)
assert kaprekar_algorithm(9) == (0, 1, None)
