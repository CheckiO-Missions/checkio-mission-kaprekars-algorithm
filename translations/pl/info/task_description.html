Weźmy losową dodatnią liczbę całkowitą (np. 5914) i zastosujmy do niej Kaprekar's routine: Najpierw najpierw tworzy Pan dwie nowe liczby całkowite, porządkując cyfry w kolejności malejącej (9541) i rosnącej (1459). Następnie proszę odjąć mniejszą liczbę od większej (9541 - 1459 = 8082). Proszę powtórzyć tę procedurę dla nowej liczby 8082.

Zastosowanie algorytmu Kaprekara do dowolnej dodatniej liczby całkowitej może skutkować trzema przypadkami:

  1. Osiągnięta zostaje stała liczba całkowita
    Dla 5914, sekwencja liczb wynosi: 5914, 8082, 8532, 6174, 6174, 6174, ...
    Zatem 6174 jest stałą Kaprekara odpowiadającą 5914.
  2. Osiągnięto cykl liczb całkowitych
    Przykład: 48, 36, 27, 45, 09, 81, 63, 27, 45 , 09 , 81, 63, 27, 45, 09, 81, 63, ...
    (Dwa różne kolory są używane do podświetlenia okręgu Kaprekara 27, 45, 09, 81, 63)
  3. Osiągnięto zero
    Chociaż technicznie zero jest również stałą (np. 7, 0, 0, 0, ...), matematycy nie uważają zera za stałą Kaprekara.

Być może zauważyli Państwo, że wiodące zera mogą odgrywać ważną rolę: w cyklu 27, 45, 09, 81, 63, dziewiątka musi być traktowana jako 09, a nie po prostu 9. Dla każdego wyniku pośredniego proszę rozważyć uzupełnić zerami wiodącymi do całkowitej liczby cyfr wejściowej liczby całkowitej.

Wejście: Dodatnia liczba całkowita n (0 ≤ n ≤ 999,999)

Dane wyjściowe: Krotka składająca się z trzech wartości:

  1. Liczba całkowita, która powinna być albo stałą Kaprekara dla n, albo (w przypadku cyklu) pierwszym numerem cyklu Kaprekara, który jest wprowadzany od n Kaprekara, który jest wprowadzany od n.
    Uwaga dla wyjaśnienia: Nawet jeśli obie liczby całkowite 48 i 92 prowadzą do tego samego okręgu (tj. 27, 45, 09, 81, 63), obie liczby całkowite wchodzą do tego okręgu pod różnymi numerami. Zatem Pana funkcja powinna zwrócić 27 (dla n = 48) i 63 dla n = 92.
  2. Liczba operacji Kaprekara, które muszą być zastosowane do n, aż do osiągnięcia stałej lub cyklu. Jeśli n jest już stałą Kaprekara (lub liczbą z cyklu Kaprekara), powinno to być liczone jako zero operacji.
  3. Boolean lub None określające, który przypadek wystąpił dla n: True dla przypadku A, False dla przypadku B, None dla przypadku C.

Przykłady: 

assert kaprekar_algorithm(5914) == (6174, 3, True)
assert kaprekar_algorithm(6174) == (6174, 0, True)
assert kaprekar_algorithm(48) == (27, 2, False)
assert kaprekar_algorithm(27) == (27, 0, False)
assert kaprekar_algorithm(45) == (45, 0, False)
assert kaprekar_algorithm(111) == (0, 1, None)
assert kaprekar_algorithm(9) == (0, 1, None)
